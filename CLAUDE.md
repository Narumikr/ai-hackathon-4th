# Claude Code プロジェクトガイドライン

## Just コマンドの使用

**重要**: 開発タスクの実行には`just`コマンドを使用すること。

### 基本ルール

* `justfile`に定義されたタスクが存在する場合は、必ずそれを使用すること
* 直接的なbashコマンド実行（`cd backend && uv run ...`）は避けること

### 主要コマンド

@justfile を確認すること

## Git Commit ポリシー

**重要**: すべての作業が完了した際には、必ずgit commitを実行してください。

### Commitルール

1. タスクが完了したら、変更内容を確認するため`git status`と`git diff`を実行する
2. 適切なファイルを`git add`でステージングする
3. 変更内容を説明する明確なコミットメッセージでcommitを作成する
4. コミットメッセージには以下を含める：
   - 変更の概要（1行目）
   - 必要に応じて詳細な説明
